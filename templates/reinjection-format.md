# .reinjection.md Format

This file captures the reentry state when an agent is re-invoked with
feedback. Generated by the orchestrator and injected into the agent's
prompt so it can understand what was attempted without reading source
files.

## Sections

### Attempt
- **Agent**: implementor | tester
- **Files modified**: [list of file paths]
- **Summary**: [one-line summary of what was changed]

### Feedback
[the error messages, test failures, or critique that triggered reentry]

### Current file state
For each modified file:

`path/to/file.ext`
```
[full file content or relevant excerpt]
```
